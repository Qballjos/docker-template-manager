Using Docker Template Manager

üìã Overview: This guide explains how to migrate your Unraid Docker setup from folder-based storage back to a vDisk-based setup using the Docker Template Manager web interface.

üß†Why Switch Back?

While folder mode offers flexibility, vDisk can be:

  • Easier to manage for beginners
  • Simpler for snapshot-based backups
  • Useful for systems with limited shares or SMB exposure
  • Single file simplicity for backup and migration

‚öôÔ∏èStep 1: Backup Templates

1. Access Docker Template Manager

Open your browser and navigate to http://your-unraid-ip:8889

2. Create a Backup

Before making any changes, backup all Docker templates:

  • Go to Backups tab
  • Click "Create Backup" button
  • Wait for backup completion - this saves all templates to /mnt/user/appdata/docker-template-manager/backups/

üß±Step 2: Stop Docker Service

In the Unraid Web Interface:

  • Go to Settings ‚Üí Docker
  • Set Enable Docker to No
  • Click Apply

Or via Terminal:

/etc/rc.d/rc.docker stop

üßπStep 3: Prepare vDisk

1. Create a New vDisk File

mkdir -p /mnt/user/system/docker/
truncate -s 30G /mnt/user/system/docker/docker.img

Note: Adjust size (30G) to your needs. Consider your current Docker usage.

2. Configure Docker Data Root

  • Go to Settings ‚Üí Docker ‚Üí Docker data-root
  • Set to: /mnt/user/system/docker/docker.img
  • Enable Docker and click Apply

üöÄStep 4: Restore Containers

1. Access Docker Template Manager

Navigate to http://your-unraid-ip:8889

2. Restore from Backup

  • Go to Backups tab
  • Find your backup in the list
  • Click "Restore" button next to your backup
  • Confirm the restore - this will copy all templates back to /boot/config/plugins/dockerMan/templates-user/

This will repopulate your new Docker vDisk using saved templates.

‚úÖVerification

1. Check Docker Tab

Go to the Docker tab in Unraid WebGUI

2. Verify Containers

  • Ensure all containers reappear in the Docker interface
  • Start each container and test functionality
  • Check Docker Template Manager - all your templates should be listed in the Templates tab

üßπStep 5: Cleanup (Optional)

Clean Up After Migration

  • Go to Templates tab
  • Review your templates - remove any that are no longer needed
  • Delete unused templates using the individual delete buttons
  • Verify all containers are working correctly

üíæBackup Strategy

After migration, continue using Docker Template Manager for ongoing management:
  • Regular backups: Use the "Create Backup" button in the Backups tab
  • Automatic backups: Set up a cron job or use Unraid's built-in backup tools
  • vDisk snapshots: Use Unraid's built-in snapshot features for the vDisk file

üß©Troubleshooting

Issue Possible Fix
Docker won't start Check vDisk file path and permissions
Missing templates Use Docker Template Manager to restore from backup
vDisk too small Increase size: truncate -s 50G /mnt/user/system/docker/docker.img
Templates not showing Check /boot/config/plugins/dockerMan/templates-user/ directory
