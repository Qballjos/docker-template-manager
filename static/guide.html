üß© What This Is About

Older Unraid setups stored Docker data inside a single large image file (docker.img, often called the vDisk).
Newer Unraid versions support a folder-based layout, which is faster, more transparent, and easier to manage.

Migrating from vDisk to folder mode provides:

  • üöÄ Faster performance
  • üíæ No wasted disk space (no fixed 20‚Äì40 GB image)
  • üîß Easier debugging and backup
  • üß± Less risk of corruption

This guide explains how to migrate your setup safely, using Docker Template Manager (DTM) to automate backups, restores, and cleanup.

‚öôÔ∏è Requirements

Before starting, make sure you have:

  1. ‚úÖ Unraid 6.12 or newer (folder support)
  2. ‚úÖ Docker service enabled
  3. ‚úÖ DTM installed and running
    (Check your WebUI port and API key if enabled)
  4. ‚úÖ A backup of your current templates and configurations

üß∞ Step 1 ‚Äì Backup Your Current Setup

Option 1: Using DTM (Recommended)

  1. Open DTM WebUI
  2. Go to Backups ‚Üí Create Backup
  3. Select your backup directory (default:
    /mnt/user/appdata/docker-template-manager/backups)
  4. Click Start Backup
  5. Verify your backup appears under Backups ‚Üí List
üîí Tip: You can schedule automatic cleanup of old backups by setting
AUTO_CLEANUP_ENABLED=true in your DTM container settings.

Option 2: Manual Backup

If you prefer to do it yourself:

cp -r /boot/config/plugins/dockerMan/templates-user /mnt/user/backups/docker-templates

üîß Step 2 ‚Äì Prepare for Migration

‚ö†Ô∏è Important: This process will temporarily stop all Docker containers. Plan accordingly!
  1. Stop all Docker containers (Main ‚Üí Docker ‚Üí Stop All)
  2. Note your current Docker settings:
    • Docker image location
    • Container templates
    • Custom networks
  3. Verify DTM backup is complete and accessible

üöÄ Step 3 ‚Äì Perform Migration

  1. Go to Settings ‚Üí Docker
  2. Change "Docker vDisk location" to "Use folder"
  3. Set folder path: /mnt/user/docker/
  4. Click Apply
  5. Wait for migration to complete (this may take several minutes)
‚úÖ Success: Unraid will automatically migrate your containers and templates to the new folder structure.

‚úÖ Step 4 ‚Äì Verify Migration

  1. Check Docker is running: Main ‚Üí Docker should show "Running"
  2. Verify containers: All your containers should be listed
  3. Test DTM:
    • Open DTM WebUI
    • Check Templates tab shows all templates
    • Verify Containers tab shows all containers
  4. Start a test container to ensure everything works

üßπ Step 5 ‚Äì Cleanup

‚ö†Ô∏è Wait at least 24 hours before cleaning up the old vDisk file to ensure everything is working correctly.
  1. Verify everything is working for at least 24 hours
  2. Delete old vDisk file:
    rm /mnt/user/system/docker/docker.img
  3. Update DTM settings if needed (usually not required)
  4. Create a new backup with DTM to verify the new setup

üÜò Troubleshooting

Containers won't start

  • Check Docker logs: docker logs [container-name]
  • Verify template paths in DTM
  • Restore from DTM backup if needed

DTM can't find templates

  • Verify template directory path in DTM settings
  • Check file permissions: ls -la /boot/config/plugins/dockerMan/templates-user
  • Restart DTM container

Migration failed

  • Restore from DTM backup
  • Check Unraid logs: Tools ‚Üí System Log
  • Contact Unraid support if needed
üí° Pro Tip: Keep your DTM backups for at least a week after migration to ensure you can restore if needed.
