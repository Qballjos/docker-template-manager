🎉 NEW FEATURE: CLONE TEMPLATES
═══════════════════════════════════════════════════════════

✨ What You Can Do:
  • Click "📋 Clone" button on any template
  • Enter new name (auto-suggests "name-copy")
  • Instant duplicate created
  • Original stays intact
  • Perfect for testing variations

═══════════════════════════════════════════════════════════

📍 WHERE TO FIND IT
═══════════════════════════════════════════════════════════

Templates Tab:
  ┌──────────────────────────────────────────────┐
  │ template.xml  [📋 Clone] [🗑️ Delete]        │
  └──────────────────────────────────────────────┘
                      ↑
                  NEW BUTTON!

═══════════════════════════════════════════════════════════

💡 HOW TO USE
═══════════════════════════════════════════════════════════

1. Go to Templates tab
2. Find template you want to clone
3. Click "📋 Clone" button
4. Dialog appears: "Clone 'my-app.xml' as:"
5. Default name: "my-app-copy" (you can change it)
6. Click OK
7. Done! New template created instantly

═══════════════════════════════════════════════════════════

🔐 SECURITY FEATURES
═══════════════════════════════════════════════════════════

✓ Validates filename (no ../  or special chars)
✓ Checks if name already exists
✓ Auto-adds .xml extension if missing
✓ API key authentication required
✓ Safe file operations
✓ Error handling for all cases

═══════════════════════════════════════════════════════════

🎯 USE CASES
═══════════════════════════════════════════════════════════

1. TESTING
   Clone "my-app" → "my-app-test"
   Test changes without affecting original

2. BACKUP BEFORE EDIT
   Clone "template" → "template-backup"
   Keep safe copy before modifying

3. CREATE VARIATIONS
   Clone "app" → "app-v2"
   Try different configurations

4. QUICK DUPLICATE
   Clone "service" → "service-2"
   Run multiple instances

5. TEMPLATE LIBRARY
   Clone common templates
   Customize for different needs

📦 TECHNICAL DETAILS
═══════════════════════════════════════════════════════════

Backend:
  • New endpoint: POST /api/templates/{filename}/clone
  • Body: { "new_name": "new-template-name" }
  • Returns: { "success": true, "filename": "..." }
  • Security: Full validation + authentication

Frontend:
  • New handler: handleCloneTemplate()
  • Clone button in each template row
  • Native prompt for name input
  • Loading state during operation
  • Success/error alerts

Files Modified:
  • app.py (+55 lines) - Backend endpoint
  • static/app.jsx (+45 lines) - Frontend handler
  • static/app.css (+53 lines) - Button styles

🔄 HOW TO UPDATE
═══════════════════════════════════════════════════════════

  2. Force Update (or stop/remove/reinstall)

  4. Test clone button

Or rebuild locally:
  cd ~/Documents/GitHub/wordpress/docker-template-manager
  git pull
  docker-compose down
  docker-compose build --no-cache
  docker-compose up -d

📊 ERROR MESSAGES
═══════════════════════════════════════════════════════════

You Might See:
  • "Invalid filename" - Used invalid characters
  • "Template already exists" - Name is taken
  • "Template name cannot be empty" - No name entered
  • "Failed to clone template" - Server error
  • "Source template not found" - Original deleted

All errors are user-friendly and actionable!
